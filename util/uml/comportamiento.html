Diagramas de comportamiento de UML Estos diagramas de comportamiento de UML se emplean para visualizar, especificar, construir y documentar los aspectos dinámicos de un sistema. Los aspectos dinámicos de un software involucran cosas tales como el flujo de mensajes y el movimiento físico de componentes en una red.
(Tomado de: El lenguaje unificado de modelado. UML 2.0. Booch 2006)

Diagramas de comportamiento
Diagrama de casos de uso: Representa un conjunto de casos de uso y actores con sus relaciones. Los diagramas de casos de uso son especialmente importante para organizar y modelar el comportamiento de un sistema.
Diagrama de secuencia:Un diagrama de secuencia es un diagrama de interacción que resalta el orden temporal de los mensajes. Presenta un conjunto de roles y sus mensajes enviados.
Diagrama de estados: Un diagrama de estados representa una máquina de estados, constituida por estados, transiciones, eventos y actividades. Son ideales para modelar el comportamiento de una clase.
Diagrama de actividades: Un diagrama de actividades muestra el flujo paso a paso. Una actividad muestra un conjunto de acciones, el flujo secuencial de cada acción. Son especialmente importantes para modelar la función de un sistema.
