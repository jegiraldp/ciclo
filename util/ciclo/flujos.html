Flujos básicos del ciclo de vida del software: El flujo describe como las actividades básicas y tareas específicas del ciclo de vida son organizadas con respecto al tiempo. De acuerdo con el tipo de solución y su alcance, se puede tomar o no un flujo en específico.

Flujo lineal: El orden de las actividades es secuencial, el resultado de una actividad sirve como insumo para la siguiente. Es necesario terminar totalmente la actividad para continuar. No es posible regresar a la actividad anterior. Solo realiza un recorrido una vez por cada actividad.
Flujo iterativo:Se caracteriza por realizar repeticiones de una o varias actividades. Este tipo de flujo permite que se evalúen los resultaos de las actividades y sea posible ejecutar de nuevo dicha actividad. Las iteraciones se realizan entre grupos de actividades o para una sola.
Flujo evolutivo: Este flujo realiza uno o varios ciclos recorriendo cada una de las actividades solo una vez. Como resultado de cada ciclo, se obtiene un incremento en las funciones del producto, las cuales pueden ser entregadas al usuario para su retroalimentación.
Flujo en paralelo: Las actividades del ciclo pueden ser ejecutas al mismo tiempo. Por ejemplo es posible realizar levantamiento de requisitos y planeación al mismo tiempo. Este diseño de flujo permite la ejecución de tareas concurrentes y distribuidas, donde no exista dependencia entre las actividades.
